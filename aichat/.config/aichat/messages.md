**Introduction to SSH**
========================

SSH (Secure Shell) is a protocol used to securely access and manage remote computers over a network. It provides a secure way to log in to a remote system, execute commands, and transfer files.

**How SSH Works**
-----------------

Here's a step-by-step explanation of the SSH process:

1. **Initialization**: When a user initiates an SSH connection, the client (usually a terminal or command prompt) sends a request to the server to establish a connection.
2. **Key Exchange**: The client and server negotiate the encryption keys to be used for the session. This is typically done using public-key cryptography, where the client and server exchange public keys and use them to create a shared secret key.
3. **Authentication**: The client authenticates itself to the server using a username and password or public key. If the authentication is successful, the server grants access to the client.
4. **Session Establishment**: Once authenticated, the client and server establish a secure session. This involves setting up the encryption and compression algorithms to be used for the session.
5. **Data Transfer**: The client can now send commands to the server, which executes them and returns the results. The data is encrypted and compressed before transmission.
6. **Session Termination**: When the client is finished, it sends a termination request to the server, and the session is closed.

**Encryption and Authentication**
-----------------------------

SSH uses a combination of encryption and authentication techniques to ensure the security of the connection:

* **Encryption**: SSH uses symmetric encryption algorithms (such as AES) to encrypt the data being transmitted. The encryption keys are exchanged during the key exchange phase.
* **Public-Key Cryptography**: SSH uses public-key cryptography (such as RSA) to authenticate the client and server. The client and server exchange public keys, which are used to create a shared secret key.
* **Digital Signatures**: SSH uses digital signatures (such as DSA) to verify the authenticity of the client and server.

**SSH Protocols**
-----------------

There are two main SSH protocols:

* **SSH-1**: This is the original SSH protocol, which is now considered insecure and obsolete.
* **SSH-2**: This is the current SSH protocol, which provides improved security and features.

**SSH Tools and Clients**
-------------------------

Some popular SSH tools and clients include:

* **OpenSSH**: A widely-used open-source SSH client and server.
* **PuTTY**: A popular SSH client for Windows.
* **ssh-keygen**: A tool for generating SSH keys.

**Example Use Case**
-------------------

Here's an example of using SSH to connect to a remote server:
```bash
# Generate a public-private key pair
ssh-keygen -t rsa

# Copy the public key to the remote server
ssh-copy-id user@remote-server

# Connect to the remote server using SSH
ssh user@remote-server
```
In this example, we generate a public-private key pair using `ssh-keygen`, copy the public key to the remote server using `ssh-copy-id`, and then connect to the remote server using `ssh`.

**SSH (Secure Shell)**

SSH (Secure Shell) is a cryptographic network protocol that enables secure remote access to a computer or server over an insecure network, such as the internet. It provides a secure way to access, manage, and control remote systems, while protecting against unauthorized access and data interception.

**Key Features of SSH:**

1. **Encryption**: SSH encrypts all communication between the client and server, ensuring that data remains confidential and secure.
2. **Authentication**: SSH uses various authentication methods, such as password, public key, or keyboard-interactive, to verify the identity of the user and prevent unauthorized access.
3. **Secure File Transfer**: SSH allows secure file transfers between systems using protocols like SFTP (Secure File Transfer Protocol) and SCP (Secure Copy).
4. **Remote Execution**: SSH enables users to execute commands remotely on a server, without exposing the underlying system's architecture.

**How SSH Works:**

1. **Client-Server Architecture**: An SSH client connects to an SSH server, which is typically running on a remote system.
2. **Handshake**: The client initiates a handshake with the server, exchanging public keys and establishing a secure connection.
3. **Authentication**: The client authenticates with the server using a chosen authentication method (e.g., password or public key).
4. **Session Establishment**: Once authenticated, the client and server establish a secure session, enabling remote access and command execution.

**SSH Clients and Servers:**

Some popular SSH clients and servers include:

* OpenSSH (server and client): Open-source, widely used, and the default SSH implementation on many Linux distributions.
* SSH Secure Shell (client): Commercial, feature-rich client with advanced functionality.
* PuTTY (client): Free, lightweight, and popular Windows-based SSH client.

**SSH Commands:**

Some common SSH commands include:

* `ssh <username>@<hostname>`: Connect to a remote system using SSH.
* `ssh localhost`: Connect to the local system (same as `ssh <current username>@localhost`).
* `scp <file> <username>@<hostname>:<path>`: Securely copy a file between systems.
* `sftp <username>@<hostname>`: Securely access a remote server's file system.

**Best Practices:**

1. **Use strong passwords**: Choose unique, complex passwords for all SSH accounts.
2. **Use public key authentication**: Enable public key authentication for secure and convenient access.
3. **Keep your SSH client and server up-to-date**: Regularly update your SSH software to ensure you have the latest security patches.
4. **Use SSH tunnels**: Encrypt traffic between systems by establishing an SSH tunnel.

By following these guidelines and using SSH responsibly, you can securely access and manage remote systems, ensuring the confidentiality, integrity, and availability of your data.

Here's a simple bubble sort function in C++ that sorts an array of integers in ascending order:

```cpp
#include <iostream>
using namespace std;

void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        // Flag to optimize - if no swaps occur, array is sorted
        bool swapped = false;
        
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap elements
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        
        // If no swapping occurred, array is already sorted
        if (!swapped) {
            break;
        }
    }
}

// Function to print array
void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
}

// Example usage
int main() {
    int arr[] = {64, 34, 25, 12, 22, 11, 90};
    int n = sizeof(arr) / sizeof(arr[0]);
    
    cout << "Original array: ";
    printArray(arr, n);
    
    bubbleSort(arr, n);
    
    cout << "Sorted array: ";
    printArray(arr, n);
    
    return 0;
}
```

This implementation:
1. Uses the bubble sort algorithm which repeatedly steps through the list
2. Compares adjacent elements and swaps them if they're in the wrong order
3. Optimizes by stopping early if the array becomes sorted before all passes complete
4. Has O(n¬≤) time complexity in worst case, O(n) in best case (when array is already sorted)

For better performance with larger datasets, you might prefer more efficient algorithms like quicksort or mergesort, but bubble sort is simple to understand and implement.

how to check storage using cli in fedora

To check storage using the command-line interface (CLI) in Fedora, you can use several commands. Here are some of the most common ones:

1. **df command**: The `df` command stands for "disk free" and is used to display the amount of disk space used and available on Linux systems.

   ```
   df -h
   ```
   The `-h` option stands for "human-readable" and makes the output easier to understand by displaying sizes in bytes, kilobytes, megabytes, and gigabytes.

2. **du command**: The `du` command stands for "disk usage" and is used to estimate the disk usage of files and directories.

   ```
   du -sh /path/to/directory
   ```
   The `-s` option stands for "summarize" and displays the total size of the directory. The `-h` option makes the output human-readable.

3. **lsblk command**: The `lsblk` command is used to display information about block devices (disks) in a tree-like format.

   ```
   lsblk
   ```
   or
   ```
   lsblk -d -o NAME,FSTYPE,SIZE,MOUNTPOINT,LABEL
   ```
   This command displays the device name, file system type, size, mount point, and label of each disk.

4. **fdisk command**: The `fdisk` command is used to display and manipulate disk partitions.

   ```
   sudo fdisk -l
   ```
   The `-l` option stands for "list" and displays the partition tables of the disks.

5. **ncdu command**: The `ncdu` command is a disk usage analyzer that displays the size of directories and files in a tree-like format.

   ```
   ncdu
   ```
   or
   ```
   ncdu /path/to/directory
   ```
   This command displays the size of each directory and file in the specified directory.

These commands can help you check the storage usage on your Fedora system.
